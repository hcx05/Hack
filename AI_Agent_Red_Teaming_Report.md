

## 一、 三模型架構設計
1. **策略大腦 (Attack Planner)**：負責高階邏輯分析。判斷受害者目前的狀態，並決定下一步的攻擊策略（如：繞過護欄、執行注入、或橫向移動）。
2. **生成大腦 (Payload Generator)**：負責戰術執行。根據 Planner 的指令，將抽象策略轉化為具備對抗性的惡意語句或代碼。
3. **受害者 Agent (Victim Agent)**：作為攻擊目標。模擬企業級 RAG 應用環境，具備長上下文處理能力與工具調用（Tool Use）權限。

## 二、 三模型選擇

| 角色 | 推薦模型 | 選擇理由 |
| :--- | :--- | :--- |
| **Planner** | Qwen-2.5-3B-Instruct | 資源優化與邏輯精準：佔用顯存極小（約 3.5GB），但 JSON 格式遵循度極高，確保自動化流程不因格式錯誤中斷。 |
| **Payload Gen** | Llama-3.1-8B-Instruct | 1. 指令遵循度高：標準版模型對於複雜格式的控制力遠勝 Abliterated。2. 對抗性潛力：8B 模型具備足夠的語言變換能力，能執行複雜的混淆（Obfuscation）任務。3. 社群基準：Llama 3.1 是目前研究 AI 攻擊鏈最通用的基準模型。 |
| **Victim** | Llama-3.1-8B-Instruct | 產業基準：作為目前主流企業 Agent 的底座，其防禦機制具備代表性，研究結果更具學術說服力。 |

## 三、 策略腦與 Payload 生成分離之理由
將攻擊代理人拆分為「策略」與「生成」兩個獨立模組，具備以下學理與工程必要性：
1. **避免同源邏輯偏差**：若決策與生成由同一模型負責，容易陷入單一思維路徑。引入異質模型（Qwen + Llama）可增加攻擊策略的多樣性與普適性。
2. **防止語意污染與角色混淆**：分離架構能確保「策略思考（冷靜、邏輯）」與「載荷生成（對抗、創造）」在獨立的對話緩存中運作，避免內部邏輯洩漏給受害者。
3. **顯存動態管理優勢**：Planner (3B) 運算快速且節省空間，讓系統能預留更多顯存給 Victim Agent 進行長上下文的 RAG 檢索，防止多輪對話時發生 VRAM 溢出（OOM）。
